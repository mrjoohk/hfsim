# Problem Statement

## Problem Background
월드모델 학습을 위한 물리기반 임무환경 시뮬레이터가 아직 없거나 학습 친화적으로 정리되어 있지 않아, 6-DoF 비행 동역학과 분기 가능한 snapshot 기반 rollout을 유지하면서도 충분한 시간 배속과 병렬 rollout 성능을 제공하는 환경을 새로 설계해야 한다. 이번 단계의 핵심 목적은 완전한 전장 디지털트윈 구현이 아니라, 월드모델이 안정적으로 동역학을 학습하고 예측 오차를 줄일 수 있는지 빠르게 검증하는 것이다.

## Scope of Impact
- System: 단일 머신 기반 월드모델 학습용 물리기반 임무환경 시뮬레이터, 시나리오 생성기, 구조화 관측 스택
- Team / Stakeholders: 시뮬레이션 환경 개발자, 월드모델 학습 개발자, 강화학습/평가 담당자
- Schedule Impact: 초기 아키텍처 결정이 이후 IF/UF 설계와 구현 방향을 고정하므로 선행 설계의 정확성이 높아야 하며, 학습 검증용 MVP를 우선 확보해야 한다

## Initial Constraints
- 단일 머신 기준으로 동작해야 한다.
- GPU가 기본적으로 장착된 PC를 우선 대상으로 하되, PC 사양에 따라 자원을 동적으로 할당해야 한다.
- 시뮬레이션 시간 배속은 기준 시나리오에서 활성화되는 전체 운동모델 스택 기준으로 실시간 대비 최소 `60x`를 목표로 한다.
- 비행체 내부 운동모델은 `6-DoF`를 사용해야 한다.
- 시뮬레이터는 임의 시점 snapshot 저장, 복원, control reinjection, branch rollout을 지원해야 한다.
- 기본 관측은 `상태벡터 + 단순 지형/위협 feature`여야 한다.
- 추후 이미지/가상 센서를 추가할 수 있도록 확장 가능해야 한다.
- 멀티에이전트는 기본 `4기`부터 시작한다.
- 포토리얼 렌더링, HIL/SIL, 유인기 통제권 전환, 실측 데이터 보정은 이번 단계 범위에서 제외한다.
