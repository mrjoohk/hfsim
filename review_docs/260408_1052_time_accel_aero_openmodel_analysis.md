# 시간 배속 기준 및 오픈 공력모델 활용 검토 메모

## 요청 요약
- `60x` 성능 기준은 6-DoF 운동모델만이 아니라 시뮬레이션 환경을 구성하는 운동모델 전체에 대해 명시되어야 한다.
- JSBSim 등 오픈 모델을 참고해 공력 계수 튜닝이 가능한지 검토한다.

## 검토 결과

### 1. `60x` 기준 해석
- 사용자의 해석이 맞다.
- 성능 기준은 "개별 모듈이 아니라 기준 시나리오에서 활성화되는 전체 운동모델 스택"에 대해 적용되어야 한다.
- 즉 아래가 모두 포함된 환경 전체 기준으로 정의하는 것이 적절하다.
  - 아군 비행체 6-DoF 동역학
  - 적/위협 기동 모델
  - 표적/장애물/환경 전이 모델
  - snapshot/restore 및 control reinjection으로 인해 필요한 상태관리 오버헤드
- 권장 문구:
  - "기준 시나리오에서 활성화되는 모든 운동모델을 포함한 시뮬레이션 환경 전체는 실시간 대비 최소 60x 이상 시간 배속을 달성해야 한다."

### 2. JSBSim 등 오픈 모델 활용 가능성
- 결론: 가능하다. 다만 "정답 모델"이 아니라 "초기화/보정/회귀 기준"으로 쓰는 것이 적절하다.

### 3. 활용 가능한 방식
- 초기 공력 계수 seed:
  - JSBSim aircraft XML이나 공개된 비행역학 파라미터를 초기 계수 후보로 사용
- 구조 참조:
  - 계수 테이블 구성 방식, 상태/제어 입력 분해 방식, 파일 기반 파라미터 체계를 참고
- 회귀 기준:
  - 동일 입력에서 속도/자세/응답 추세를 비교하는 sanity check oracle로 활용
- 캘리브레이션 케이스 생성:
  - 고도, 속도, 받음각, 조종 입력별 대표 case를 만들어 우리 모델의 튜닝 목표값으로 사용

### 4. 주의할 점
- JSBSim 모델은 기체별 품질 편차가 크다.
- 공개 모델은 특정 기체/비행영역에만 유효할 수 있다.
- 우리 시뮬레이터는 학습 효율을 위해 단순화된 6-DoF를 쓰므로, JSBSim과 1:1 수치 일치를 목표로 두면 오히려 과도한 복잡도가 생길 수 있다.
- 라이선스와 데이터 출처 기록은 남겨야 한다.

### 5. 권장 반영 방식
- 이번 단계 요구사항에는 아래 수준으로 반영하는 것이 적절하다.
  - 오픈 공력모델 또는 공개 파라미터 자료를 입력으로 받을 수 있는 calibration workflow 지원
  - 오픈 모델 기반 초기 계수 import 또는 비교 리포트 지원
  - 실기체급 정확도 보장은 이번 단계 범위 밖으로 유지

## 판단 근거
- 사용자는 정밀 공력계수 튜닝을 가능하면 포함하고 싶다고 명시했다.
- 그러나 현재 단계 목표는 world-model 학습 검증이므로, 오픈 모델은 고정밀 정답원보다는 튜닝 bootstrap 자산으로 쓰는 것이 비용 대비 효과가 높다.
